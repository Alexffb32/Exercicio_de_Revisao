1. IDE (Ambiente de Desenvolvimento Integrado):


O que é uma IDE?

Uma IDE é um Ambiente de Desenvolvimento Integrado, uma ferramenta que fornece recursos abrangentes para facilitar o desenvolvimento de software, unificando diversos aspectos do ciclo de vida do desenvolvimento em uma única interface.


Cite pelo menos três funcionalidades essenciais de uma IDE:

Editor de código com realce de sintaxe. ; Depurador (debugger) para identificação e correção de erros. ; Gerenciamento de projetos e integração com sistemas de controle de versão.

2. Git:


Explique o propósito do sistema de controle de versão Git:

O Git é um sistema de controlo de versão distribuído projetado para rastrear alterações em código-fonte durante o desenvolvimento de software, possibilitando colaboração eficiente, controle de versões e histórico detalhado.


Descreva as diferenças entre "commit" e "push" no contexto do Git:

Um "commit" registra alterações localmente no repositório. ; Um "push" envia esses commits para o repositório remoto, compartilhando as alterações com outros colaboradores.

3. GitHub:


Qual é a função principal do GitHub?

O GitHub é uma plataforma de hospedagem de repositórios Git, facilitando o armazenamento, colaboração e controle de versão de projetos de software.


Como o GitHub facilita o trabalho colaborativo em projetos de software?

Facilita colaboração através de recursos como controle de acesso, problemas (issues), pull requests, e facilidade de revisão de código.

4. Branches em Git:


O que são branches (ramificações) em Git?

Branches são ramificações independentes do código principal, permitindo o desenvolvimento paralelo de funcionalidades sem afetar a versão principal.


Por que seria útil criar e mesclar branches em um projeto?

Facilita o desenvolvimento simultâneo de funcionalidades, correções de bugs ou experimentações, mantendo o código principal estável.

5. Merge e Pull Request:


Explique o processo de realizar um merge em Git:

Merging envolve a incorporação de alterações de uma branch para outra, combinando as modificações.


Qual é o propósito de um Pull Request (PR) no GitHub?

Um Pull Request é uma solicitação para mesclar alterações de uma branch para outra, permitindo revisões, discussões e testes antes da incorporação.

6. Deploy:


Defina o termo "deploy" no contexto de desenvolvimento de software:

Deploy refere-se à implantação de uma aplicação em um ambiente específico, como produção, tornando-a acessível aos usuários finais.


Mencione alguns métodos comuns de realizar o deploy de uma aplicação:

Exemplos incluem deploy manual, integração contínua (CI/CD), containers (como Docker), e plataformas de nuvem.

7. Ambientes de Desenvolvimento, Teste e Produção:


Por que é importante ter ambientes distintos durante o ciclo de vida de desenvolvimento?

Ambientes distintos garantem testes adequados antes de uma aplicação ser implantada em produção, evitando problemas não detectados durante o desenvolvimento.


Como você garantiria que uma aplicação funciona corretamente antes de ser implantada em produção?

Utilizando ambientes de teste e pré-produção para realizar testes rigorosos, automação de testes, e revisões de código.

8. Controle de Versão:


Além do Git, mencione outros sistemas de controle de versão:

Exemplos incluem SVN (Subversion), Mercurial e CVS (Concurrent Versions System).


Qual é a importância do controle de versão em projetos de software?

Fornece histórico detalhado, facilita colaboração, rastreia mudanças e permite reversão a versões anteriores.

9. Fluxo de Trabalho Colaborativo:


Como o Git e o GitHub facilitam o trabalho colaborativo entre membros de uma equipe de desenvolvimento?

Permitindo a sincronização de código, controle de versão eficiente, revisões de código, gerenciamento de issues e pull requests.

10. Ferramentas de Build:


O que são ferramentas de build e qual é seu papel no desenvolvimento de software?

Ferramentas de build convertem código-fonte em executáveis ou outros artefatos. Seu papel é automatizar o processo de compilação, teste e empacotamento.


Dê exemplos de ferramentas de build utilizadas na indústria:

Exemplos incluem Maven, Gradle, Ant, e ferramentas específicas de linguagens como npm para JavaScript ou pip para Python.
